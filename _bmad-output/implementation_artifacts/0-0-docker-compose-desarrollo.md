# Story 0.0: Docker Compose de Desarrollo

Status: done

## Story

As a desarrollador,
I want un entorno de desarrollo con PostgreSQL, RabbitMQ y Mailpit que levante con un solo comando,
so that pueda empezar a codificar sin configurar infraestructura manualmente.

## Acceptance Criteria

1. PostgreSQL 15 disponible en localhost:5432 (db: sie, user: sie, password: sie_dev)
2. RabbitMQ 3.x con management UI en localhost:15672 (user: sie, password: sie_dev)
3. Mailpit SMTP mock en localhost:1025 con web UI en localhost:8025
4. Script `dev.sh` que levante los 3 servicios con un solo comando
5. Compatible con Podman (usuario rootless)
6. Backend configurado en `application-dev.properties` apuntando a estos servicios
7. Maven wrapper (`mvnw`) funcionando para backend

## Dev Notes

### Architecture Patterns
- Todos los servicios corren en un `podman pod` compartiendo network namespace
- RabbitMQ requiere `podman run` directo — podman-compose tiene bug de permisos con erlang.cookie
- Mailpit acepta conexiones SMTP sin TLS en dev (`MP_SMTP_AUTH_ALLOW_INSECURE=true`)

### Files Created
- `docker-compose.yml` — referencia, pero dev.sh usa podman pod directamente
- `dev.sh` — script principal con soporte Podman/Docker
- `backend/application-dev.properties`
- `backend/.mvn/wrapper/`
- `backend/mvnw`, `backend/mvnw.cmd`

### Verification
```bash
./dev.sh start    # Levanta PostgreSQL + RabbitMQ + Mailpit
./dev.sh status   # Verifica que los 3 estén corriendo
./dev.sh stop     # Detiene y limpia
```
